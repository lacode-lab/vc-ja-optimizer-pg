# vc-ja-optimizer-pg — 音声入力テキスト最適化

音声入力で得られた日本語テキストを修正・整形し、クリップボードにコピーして表示するスキル。

## 入力

`$ARGUMENTS` に音声入力されたテキストが渡される。空の場合は、修正したいテキストを貼り付けるようユーザーに依頼して終了する。

## 手順

### 1. 変換辞書の読み込み

以下の2つの辞書を読み込む（存在するものだけでよい）:

1. **共通辞書**: このスキルと同じディレクトリにある `dictionary.md`
2. **プロジェクト辞書**: カレントプロジェクトの `.claude/vc-ja-dictionary.md`（存在する場合のみ。プロジェクト特有の用語はこちらが優先）

辞書は `| 誤変換・音声認識結果 | 正しい表記 |` 形式のMarkdownテーブル。入力テキスト中に「誤」列の表現（および明らかにその音訳・類似表記と分かるもの）があれば「正」列の表記に置き換える。

### 2. テキストの修正

辞書変換に加えて、以下の観点で文章を修正する:

- 同音異義語の誤変換の修正（文脈から正しい漢字を判断）
- 音声認識特有の誤認識の修正（助詞の脱落・重複、無意味なカタカナ語など）
- フィラーの除去（「えーと」「あのー」「まあ」など）
- 句読点の適切な補完・整理
- 明らかな言い直し・重複表現の整理

**重要**: 意味・意図・トーンは変えない。内容の追加・要約・言い換えはしない。あくまで「話者が入力しようとした正しいテキスト」への復元に徹する。

### 3. 修正結果の表示（必須・省略禁止）

確認を求める**前に**、修正後のテキストを**句点（。）ごとに改行した形**（丸改行）で、通常のメッセージとして必ず表示する。例:

```
この内容で命令をClaude Codeに送るのか、それともこの命令をプロンプトに指示するのかを尋ねるだけのアプリです。
さらに、変換辞書を備えており、プロジェクト特有のものや間違いやすいものは、その変換辞書を元に変換されます。
```

あわせて、辞書や文脈による変換を行った場合は「クロードコード → Claude Code」のように変換箇所を簡潔に列挙する。**このステップを飛ばして手順5の確認に進んではならない**（ユーザーは内容を見ないと判断できない）。

### 4. クリップボードへのコピー

修正後のテキスト（丸改行加工していない元の形）を `pbcopy` にヒアドキュメントで直接渡してコピーする。`cat` やパイプは使わないこと（余計な承認プロンプトが出るため）:

```bash
pbcopy <<'EOF'
（修正後テキスト）
EOF
```

`pbcopy` が存在しない環境（macOS以外）では、コピーをスキップして表示のみ行い、その旨を一言添える。

### 5. 締めの案内（実行はしない）

最後に次の2点を伝えて、**そのままターンを終了する**:

- クリップボードにコピーした旨
- 「この内容で実行する場合は、そのまま貼り付けて送信してください」という案内

修正後テキストを命令として**このスキル内で実行してはならない**。実行するかどうかの確認（AskUserQuestion等）も行わない。

> **注釈: なぜ確認して実行する方式にしないのか**
> このスキルは `model: sonnet` で動くため、スキル内で確認して続けて実行すると、本題の作業までSonnetで実行されてしまう。表示のみで終了し、ユーザーがコピー済みテキストを新しいプロンプトとして送信すれば、セッションのメインモデルで実行される。
